Chat seguro con monitorización

Escribe una consulta, acción o instrucción y el sistema revisa si el mensaje o la respuesta del asistente contienen un riesgo de seguridad. El objetivo es detectar intentos de inyección, escalada de privilegios o fugas de datos.

Cómo funciona
  • Se crea una sesión local en el navegador.
  • Se analiza primero el texto del usuario (Input) con reglas de detección.
  • Se consulta el modelo Ollama para generar la respuesta.
  • Si no hay coincidencia en Input, se revisa el texto de salida (Output).
  • Con una coincidencia en cualquiera de los dos frentes ya se marca incidente.
  • Se registra el historial y los incidentes en SQLite y JSONL.
